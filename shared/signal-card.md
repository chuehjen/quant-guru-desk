# Signal Card — 结构化结论信号

每次分析都以一张**短结论**收尾——四行，几秒钟读完。完整的证据分级 signal card 默认折叠在 `<details>` 块里，需要时再展开。

> **设计意图：** 普通用户的第一性问题是"能不能买、会亏多少"。给他们四行就够。专业用户问"证据是什么、怎么打分"，他们会主动展开 details。一份输出同时服务两类人。

## 默认输出（必填，永远先出现）

```
---
**📊 [GURU] 一句话观点：** [≤30字，给出方向 + 关键论据]
**信心度：** [1-5] ⭐
**建议仓位：** [新建仓 X% / 维持现有 / 不建仓]
**最大风险（最强反方）：** [一句话，认为这笔交易不该做的最强理由]
---
```

四行原则：

1. 一句话观点必须**有方向**——不能是"值得关注"这种虚词。要写"NVDA 现价低于 30 天前卖出位 + 板块刚反弹，逢弱补"。
2. 信心度严格遵守 1-5 校准（见下文），不允许"3.5"。
3. 仓位三选一——具体数字 / "维持" / "不建仓"。模糊不行。
4. **最大风险/最强反方**——必须是**会让方向翻转**的因素，且必须以**反方视角**写：
   - ❌ 废话："市场整体下跌"、"宏观波动"、"政策风险"
   - ❌ 自我安慰式："如果支撑位跌破才止损"（这是止损不是反方）
   - ✅ 反方视角："Chanos 会指出 AI 资本支出周期顶部已现，CoWoS 产能 2026Q4 将过剩"
   - ✅ 反方视角："Buffett 会问：这家公司 10 年后还在吗？答案不确定"

   **强制反方原则**：每张信号卡必须想象一位最强的反对者会怎么拆台，把他/她最致命的一击写下来。如果你想不出反方理由，说明分析不够深——回去补。

## 完整 Signal Card（可折叠）

```
<details>
<summary>📊 完整 signal card（点击展开）</summary>

| 字段 | 值 |
|------|-----|
| Ticker | [TICKER] |
| 大神 | [guru name] |
| 信心度 | [1-5] ⭐ (1=极度不确定, 3=中性, 5=高度确信) |
| 动作 | **BUY** / **HOLD** / **SELL** / **WATCH** / **AVOID** |
| 建议仓位 | [X%] of portfolio (新建仓) 或 "维持现有 X%" (已持有) |
| 入场价 | $[price] (BUY 时必填; HOLD/SELL 填当前价) |
| 止损/退出条件 | $[price] 或 [条件描述] |
| 时间框架 | [持有预期: 数天/数周/数月/长期] |
| 核心逻辑 | [一句话，≤30字] |
| **最强反方 (Bear case)** | **[反对者视角的最致命一击，1-2 句]** |
| 历史命中 | [由 track_calls.py 自动注入；首次为空] |

</details>
```

**何时默认展开**（不折叠）：

- 信心度 ≥ 4（高信心要求用户看清证据）
- 用户明确说"详细看 / 给我完整分析 / 展开"
- 反思回溯块显示同一只票上次错过的情况（强制让用户看到历史）
- 触发了红旗披露（`shared/evidence-standards.md`）

其他情况一律折叠。

## 字段说明

### 信心度 (Conviction 1-5)

不是对股票看涨看跌的程度，而是**对自己判断的确信程度**：

- **5** — 所有证据一致指向同一方向，高度确信。罕见，一年出现 2-3 次。
- **4** — 强证据支持，少量不确定性。可以用较大仓位。
- **3** — 证据分歧或数据不足，判断有合理的另一面。保守仓位。
- **2** — 方向模糊，更多是"倾向"而非"确信"。最小仓位或仅观察。
- **1** — 几乎是猜测。不建议行动，仅记录观点供后续验证。

**信心度直接影响 `calls.json` 的 conviction-weighted score**：5 分对了 +5、错了 -5；2 分对了 +2、错了 -2。所以高信心是有惩罚的——不要乱标。

### 动作 (Action)

| 动作 | 含义 | 何时使用 |
|------|------|---------|
| **BUY** | 当前是合理买入时机 | 估值/时机/宏观三者至少两个亮绿灯 |
| **HOLD** | 已持有，继续持有 | 论点未变，无需加减仓 |
| **SELL** | 建议卖出（部分或全部） | 触发止损/论点崩塌/目标达成 |
| **WATCH** | 有兴趣但时机未到 | VCP 未完成 / 估值未到位 / 等催化剂 |
| **AVOID** | 明确不碰 | Stage 4 / 护城河崩塌 / 宏观逆风 |

### 各大师特殊适配

完整卡片是**统一壳**，但内容反映各自方法论：

| 大师 | 止损/退出条件 典型填法 | 时间框架 典型值 |
|------|----------------------|---------------|
| Buffett | "护城河受损 / 管理层不诚信" (无价格止损) | 长期 (5-10年) |
| Dalio | "象限切换至 [X]" 或 "配置偏离 >5%" | 数周到数月 |
| Beth Kindig | "$X (跌破前瞻营收支撑位)" | 数月 |
| Serenity | "$X (供应链逻辑断裂)" | 数月 |
| Cathie Wood | "$X 或 TAM 叙事破裂" | 数月到数年 |
| Minervini | "$X (VCP 低点下方 7-8%)" — 精确价格 | 数周 |

### 会诊模式的信号卡

Panel 会诊时，每位大师各出一张完整卡片（默认折叠），然后在 `主持人小结` 后追加一张**共识信号卡**（默认展开，因为这是用户最需要看的）：

```
## 📊 共识信号卡

| 字段 | 值 |
|------|-----|
| Ticker | [TICKER] |
| 参与大神 | [A, B, C] |
| 多数动作 | [BUY/HOLD/SELL/分歧] |
| 平均信心度 | [X.X] |
| 关键分歧 | [一句话描述最大意见不同] |
| **最强反方** | **[即使多数派看多，最有说服力的反对意见是什么]** |
| 建议 | [跟随多数 / 因分歧过大建议观望 / 按用户风险偏好选择] |
```

Panel 会写入 `calls.json` 一条 `panel-consensus` 记录（`guru: "panel-consensus"`），动作字段取多数票，信心度取平均。

## 写入 `calls.json`

每张签发的 signal card（无论默认短输出还是完整卡片）都必须**调用一次** `scripts/track_calls.py append`：

```bash
python3 scripts/track_calls.py append --json '{
  "id": "2026-06-09-minervini-NVDA-001",
  "created_at": "2026-06-09T13:30:00+08:00",
  "guru": "minervini",
  "ticker": "NVDA",
  "action": "BUY",
  "conviction": 4,
  "entry": 205.10,
  "stop": 190.00,
  "target": 250.00,
  "horizon_days": 30,
  "thesis": "VCP 3rd contraction complete, pivot $206 breakout on volume",
  "context": "Market Stage 2",
  "competition_run": false
}'
```

详细 schema 见 `shared/calls-schema.md`。**这一步不是可选——四行短输出之后立刻 append，否则 Reflection-as-core 形同虚设。**

## 何时可以省略信号卡

- **Learn 模式**（纯教学，无具体 ticker）→ 不需要短结论也不需要完整卡片
- **竞赛日报**（已有完整持仓表 + 交易计划）→ 用竞赛格式的交易计划表替代完整卡片，但**仍需在末尾放四行短结论**，并对当日每个 BUY/SELL 动作各写一行 `calls.json`。HOLD 不写 `calls.json`（减少噪音）。

```
## 📊 今日信号摘要
| Ticker | 动作 | 信心 | 止损 |
|--------|------|------|------|
| NVDA | BUY | 4 | $195 |
| MSFT | HOLD | 3 | — |
```

## 规则

1. 四行短结论永远是最后一段（在分析正文之后、免责声明之前）
2. 完整卡片在四行短结论之上，默认 `<details>` 折叠
3. 信心度必须诚实——不确定就标低。标高意味着你愿意拿最大仓位，并接受 conviction-weighted score 的惩罚
4. 止损字段对 Minervini 是**精确价格**（硬性要求），对 Buffett/Dalio 可以是条件描述
5. 动作和信心度在分析过程中可能演变——最终卡片反映分析结束后的结论，不是中途的初步倾向
6. 若数据不足以得出任何结论，信心度标 1，动作标 WATCH，并在核心逻辑写明"数据不足，待补充 [X]"。WATCH 在 `calls.json` 永远是 `neutral`，不影响 hit rate
7. **强制反方字段**——四行短结论的"最大风险/最强反方"和完整卡的"Strongest bear case"必须以**反对者视角**写。若你写不出反方理由（或只能写出"市场下跌"这类废话），视为分析不充分，必须返工。这是治"LLM 顺着用户偏好讲"的硬性约束。
